# Product

## Register

product

## Users

Developers and technical teams building automation or integrations with WhatsApp. They're in a workflow context—deploying bots, managing credentials, monitoring connection status, and integrating API keys into their applications. They need reliable tooling that gets out of the way.

## Product Purpose

A WhatsApp bot management platform that makes it simple to deploy and manage multiple bot instances without complexity. Success means developers can spin up a bot, grab an API key, and integrate it into their application within minutes. The platform handles the WhatsApp connection lifecycle (QR scanning, session management, logout) so developers can focus on their business logic.

## Brand Personality

Technical, reliable, straightforward—with touches of friendliness and accessibility. This is a developer tool first, but it shouldn't feel cold or intimidating. Professional enough to trust with production workloads, approachable enough that someone can figure it out without a manual.

## Anti-references

Avoid:
- **Overly playful consumer apps**: No gamification, no excessive animation, no emoji-heavy UI that undermines the technical nature of the work
- **Dense enterprise dashboards**: No overwhelming data tables, complex navigation hierarchies, or feature bloat that obscures the core workflow
- **Flat gray utility tools**: Not lifeless or uninspired—utility doesn't mean boring
- **SaaS cream/beige templates**: No generic warm-tinted landing page aesthetic, no decorative gradient text, no side-stripe borders on cards

## Design Principles

1. **Fast to value**: The path from login to working bot should be obvious and short. Every screen serves the core workflow.
2. **Show state clearly**: Connection status, QR availability, API key visibility—never make the user guess what's happening or what to do next.
3. **Trust through simplicity**: Reliability comes from doing fewer things well, not from adding reassurance theater. Clean interfaces and consistent feedback build confidence.
4. **Respect developer context**: Developers are likely switching between this tool and their code editor. The UI should be clear enough to parse at a glance, not demand sustained focus.

## Accessibility & Inclusion

WCAG 2.1 AA compliance as the baseline. Prioritize:
- Sufficient color contrast (≥4.5:1 for body text, ≥3:1 for large text and UI components)
- Keyboard navigation for all interactive elements
- Clear focus indicators
- Semantic HTML and proper ARIA labels where needed
- No motion-dependent interactions; respect `prefers-reduced-motion`
